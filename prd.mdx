# Zaymo API Documentation PRD

## Overview
This PRD outlines the plan to update Zaymo's API documentation to make it easy for companies to build integrations with Zaymo's in-email purchase functionality. The documentation will be built using Mintlify to create professional, easy-to-understand documentation.

## Project Goals
- Create clear, comprehensive documentation for integration partners
- Outline all required steps for integration
- Keep explanations simple and easy to understand
- Provide detailed API references with examples

## Documentation Structure

### 1. Introduction
- Overview of Zaymo's functionality
- Example of in-email purchase (with GIF)
- Case studies highlighting successful implementations
- Value proposition for integration partners

### 2. Use Cases
Detailed explanation of each supported use case:

#### 2.1 Product Upsells
- Overview of functionality
- Required endpoints and data
- Implementation steps
- Example flows

#### 2.2 Portal
- Subscription management capabilities
- Required endpoints
- Implementation steps
- Example flows

#### 2.3 Reactivate
- Cancelled subscription reactivation
- Required endpoints
- Implementation steps
- Example flows

### 3. API Reference
Detailed documentation for each endpoint:

#### 3.1 Authentication
- API key setup and usage
- Security best practices

#### 3.2 Product Upsells Endpoints
- Get Products endpoint
- Add Product endpoint
- Customer Info endpoint (optional)

#### 3.3 Portal Endpoints
- Subscription Info endpoint
- Reschedule endpoint
- Send Now endpoint

#### 3.4 Reactivate Endpoints
- List Cancelled Subscriptions endpoint
- Reactivate endpoint

### 4. Testing
- Test customer profile setup
- Integration testing process
- Troubleshooting common issues
